# neco-brep -- 数理詳細

この文書では neco-brep が採る幾何表現と、ブール演算・テセレーションまでの処理の骨格を整理する。
解析表現をどこまで保ち、どこで離散化するかを明示することを目指す。

使い方と API は [README-ja.md](README-ja.md) を参照。

前提とする対象:

- 曲面族
- プロファイル駆動生成
- 2D/3D ブール演算
- 三角形化

特に 3D ブール演算は、単一連結シェルを主対象にし、テセレーションは一様サンプリングにもとづく。

## 立体境界表現

$$
\mathrm{Shell} = (V, E, F)
$$

- $V$: $\mathbb{R}^3$ 上の頂点
- $E$: `Curve3D` で表される境界曲線
- $F$: トリムループと基底 `Surface` を持つ面

面は「基底曲面」と「その上の境界ループ」の組であり、トポロジと幾何を分けて保持する。

各面ループの向きは法線の向きを決めるので、内外判定やブール演算ではこの向きがそのまま効く。

## 解析曲面と生成曲面

neco-brep は、操作により解析曲面が維持されるならば、できるだけ長く解析表現を保つ。

高次プロファイルでは `SurfaceOfRevolution`、`SurfaceOfSweep`、NURBS 曲面として保持する。

### 平面

平面は基準点 $\mathbf{o}$ と単位法線 $\mathbf{n}$ に直交する 2 方向 $\mathbf{u}, \mathbf{v}$ を使って以下のように表す:

$$
\Pi = \{ \mathbf{o} + \alpha \mathbf{u} + \beta \mathbf{v} \mid \mathbf{u}, \mathbf{v} \perp \mathbf{n} \}
$$

線形押し出しの側面や、プロファイル生成のふた面はこの形になる。

### 円柱・円錐

円柱は軸方向 $\mathbf{a}$ と半径 $r$ を使って以下のように表す:

$$
S(\theta, t) = \mathbf{o} + t\mathbf{a} + r(\cos\theta\,\mathbf{u} + \sin\theta\,\mathbf{v})
$$

円錐では半径が軸方向に線形に変化して以下のようになる:

$$
r(t) = t \tan \phi
$$

したがって直線プロファイルの回転は軸との相対位置に応じて、平面・円柱・円錐へ分類できる。

### 球・楕円体・トーラス

球面、楕円体、トーラスは陰関数が簡潔なので、解析形のまま保持する。

- 球面: $\|\mathbf{x} - \mathbf{c}\|^2 = r^2$
- 楕円体: $\frac{x^2}{r_x^2} + \frac{y^2}{r_y^2} + \frac{z^2}{r_z^2} = 1$
- トーラス: 軸座標系へそろえた上で $(\sqrt{x^2 + y^2} - R)^2 + z^2 = r^2$

この選択により、ブール交差や法線評価の時点で不要な離散化を避けられる。

## プロファイル駆動生成

### 押し出し

押し出しは 2D プロファイル $C(v)$ を方向ベクトル $\mathbf{d}$ に沿って移す操作で以下のように表す:

$$
S(u, v) = C(v) + u\mathbf{d},
\qquad
u \in [0, 1]
$$

次数 1 のプロファイル区間は平面になる。高次区間では、実装はそのまま `SurfaceOfSweep` / NURBS 曲面として持つ。

### 回転

2D プロファイル $(r(v), z(v))$ を軸まわりに回すと以下のようになる:

$$
S(\theta, v) =
\mathbf{c}
+ z(v)\mathbf{a}
+ r(v)(\cos\theta\,\mathbf{u} + \sin\theta\,\mathbf{v})
$$

線形区間は平面、円柱、円すいへ落ちる。高次区間は `SurfaceOfRevolution` または NURBS 曲面片として保持する。

### スイープ

スイープは空間中心線 $\Gamma(u)$ と、その上を運ぶ断面座標 $(x(v), y(v))$ を使って以下のようになる:

$$
S(u, v) = \Gamma(u) + x(v)\mathbf{n}(u) + y(v)\mathbf{b}(u)
$$

と書ける。ここで $(\mathbf{t}, \mathbf{n}, \mathbf{b})$ は移動フレームである。

neco-brep は回転最小フレーム (RMF) を使い、中心線に沿った不要なねじれを抑える。

実装はスパインを Bezier 分解済み制御点列として受け取り、曲率半径が断面半径より小さくなって自己交差しやすい場合は事前に弾く。

### ロフト

ロフトは配置済み断面曲線 $C_i(v)$ を断面方向に補間して曲面を作る。straight モードでは以下の直線補完面になる:

$$
S(u, v) = (1-u)C_0(v) + uC_1(v)
$$

smooth モードでは Catmull-Rom 系の接線にもとづく 3 次補間を使い、隣接断面間で $C^1$ 連続をねらう。

実装上は、各 section が対応する Bezier スパン数を共有している必要がある。

## ブール演算

### 2D: `NurbsRegion`

2D ブール演算は、交差、分割、分類、選択、再接続の流れで進む。

1. 境界曲線どうしの交点を求める
2. 交点パラメータで曲線を分割する
3. 各分割片を、相手領域に対して内側 / 外側 / 重なりに分類する
4. `Union` / `Subtract` / `Intersect` に応じて残す片を選ぶ
5. 選んだ片をループへ縫い直す

共線重なりは重要である。単なる交点列挙だけでは重なり区間を扱えないので、実装は overlap 区間を明示的に保持し、中点と法線方向で分類する。

`boolean_2d_all` は 0 個、1 個、または複数個の結果領域を返せる。単一領域専用の `boolean_2d` は、その薄い補助関数である。

### 3D: `Shell`

3D ブール演算も骨格は同じである。

1. 面どうしの交差曲線を求める
2. 交差曲線で各面を `SubFace` に分割する
3. `point_in_shell` で各部分面を分類する
4. ブール種別に応じて残す部分面を選ぶ
5. 選んだ部分面から `Shell` を再構築する

演算の中心が「面の切断と分類」であり、最初から三角形メッシュに落とすことはしない。

## テセレーション

`Shell::tessellate` で初めて三角形メッシュへ落とす。

### 平面面

平面面は局所 2D 座標系へ射影し、制約付き Delaunay 三角形分割 (CDT) で三角形化する。境界のサンプリング点を制約辺として保つことで、輪郭を維持したまま品質のよい三角形を得る。

### 非平面面

平面以外の面はパラメータ空間で一様に分割して評価する。trim がある場合は trim ループを使って内部を切り分けるが、基本方針は曲率適応ではなく一様分割である。したがってここでの離散化誤差は density に依存する。

最後に、面ごとに得たメッシュを結合し、近接頂点を許容誤差内でまとめて水密性を改善してから出力する。
