# neco-cdt -- 数理詳細

この文書では neco-cdt が使う幾何判定と、制約 Delaunay 三角形分割の骨格をまとめる。中心になるのは、三角形分割の局所操作を進めるために「点が左にあるか右にあるか」「点が外接円の内側か外側か」を安定に判定することである。使い方と API は [README-ja.md](README-ja.md) を参照。

ここで扱う対象は 2D 制約 Delaunay 三角形分割である。実装は Bowyer-Watson による逐次挿入で通常の Delaunay 分割を作り、その後に制約辺を回復する。3D 向けの `orient3d` と `insphere` も公開しているが、本体アルゴリズムの主対象は 2D である。

## 符号つき幾何判定

三角形分割を組み立てるときに本当に必要なのは、長さや角度の精密な値そのものではない。必要になるのは、点の並び順と内外関係である。ある点が辺の左側か右側か、ある点が外接円の内側か外側かが分かれば、局所操作を正しい向きで進められる。

neco-cdt の幾何判定では、この「どちら側か」を表す符号が最も重要である。

### `orient2d`

最初の基本判定は、3 点 $a, b, c$ がどちら向きに並んでいるかである。3 点を結んだ三角形を 3D で考えると、ベクトル $\overrightarrow{ca}$ と $\overrightarrow{cb}$ の外積は三角面法線に対応する。2D ではこの法線は紙面に垂直な $z$ 方向にしか出ないので、必要なのはその $z$ 成分の符号だけである。

3 点 $a, b, c$ の向きは

$$
\begin{vmatrix}
a_x - c_x & a_y - c_y \\
b_x - c_x & b_y - c_y
\end{vmatrix}
$$

の符号で決まる。これは「$c$ から見て、$a \to b$ が左回りか右回りか」を数式にしたものであり、同時に三角面法線の向きを 2D に落とした量でもある。

これは 2D 外積

$$
(a_x - c_x)(b_y - c_y) - (a_y - c_y)(b_x - c_x)
$$

に等しく、幾何的には三角形 $(a,b,c)$ の符号つき面積の 2 倍である。正なら反時計回り、負なら時計回り、0 なら共線である。

### `incircle`

次の基本判定は、点 $d$ が三角形 $(a,b,c)$ の外接円の内側にあるかどうかである。ここで外接円とは、三角形の 3 頂点すべてを通る円を指す。

点 $d$ がその外接円の内側にあるかは

$$
\begin{vmatrix}
a_x - d_x & a_y - d_y & (a_x - d_x)^2 + (a_y - d_y)^2 \\
b_x - d_x & b_y - d_y & (b_x - d_x)^2 + (b_y - d_y)^2 \\
c_x - d_x & c_y - d_y & (c_x - d_x)^2 + (c_y - d_y)^2
\end{vmatrix}
$$

の符号で判定する。$(a,b,c)$ が反時計回りなら、正が内側、負が外側、0 が円周上である。

この判定が現れる理由は、Delaunay 条件が「外接円の内側に他の点が入らないこと」で書けるからである。したがって外接円判定は、三角形分割の局所更新を決める中心的な述語になる。

この行列式は放物面への持ち上げで理解できる。2D 点 $(x,y)$ を

$$
(x,y,x^2+y^2)
$$

へ写すと、平面上の円は放物面を平面で切った断面に対応する。したがって外接円判定は、持ち上げた点に対する 3D 向き判定へ読み替えられる。

## なぜ適応精度が必要か

ここまでの述語は式だけ見れば単純だが、数値計算としては退化近傍が難しい。`orient2d` や `incircle` を素朴な `f64` 行列式で計算すると、点がほぼ共線、またはほぼ同一円上にある場合に桁落ちが起こり、符号を取り違えることがある。三角形分割では、この符号誤りがそのまま不正な辺フリップや壊れた局所構造につながる。

neco-cdt は Shewchuk 系の適応精度判定を使う。

1. まず通常の浮動小数点演算で近似値を出す
2. 丸め誤差の上界を見積もる
3. 近似値がその上界より十分大きければ符号をそのまま返す
4. 0 をまたぐ可能性が残るときだけ、拡張演算によるより厳密な計算へ進む

多くの入力では早い段階で符号が確定するので、平均計算量は通常の浮動小数点判定に近い。一方で退化近傍では必要なだけ精度を上げられる。

## Delaunay 条件

三角形分割が Delaunay であるとは、どの三角形についても、その外接円の内側に他の頂点が入らないことである。局所的には、隣接頂点 $d$ に対して

$$
\mathrm{incircle}(a,b,c,d) \le 0
$$

が成り立つことに対応する。

この条件は、細長い三角形を避けやすいという幾何的性質と直結している。幾何的には、ある辺の向こう側の頂点が外接円の深く内側へ入り込むなら、その辺の対角線の取り方は局所的に不自然である。Delaunay 分割は、そのような不自然な対角線を避ける方向へ寄る。

有限要素法や補間で Delaunay 分割が好まれるのはこのためである。

## Bowyer-Watson による逐次挿入

neco-cdt は通常の Delaunay 分割を Bowyer-Watson 法で構築する。全体の流れは次の通りである。

### 初期化

まず入力領域を覆う十分大きな **スーパー三角形** を置く。以後の挿入はこの三角形の内部で始める。

### 点の挿入

Bowyer-Watson 法の見通しは単純である。新しい点 $p$ を入れたときに Delaunay 条件を壊す三角形だけを局所的に入れ替える。

具体的には

1. $p$ を含む外接円を持つ三角形群を集める
2. その集合を空洞として取り除く
3. 空洞境界の各辺と $p$ を結び、新しい三角形群で埋める

という手順を取る。外接円の内側に $p$ を含む三角形は、$p$ を入れた後には Delaunay 条件を満たせないので、まとめて取り除くことになる。

空洞を構成する三角形は、連結な領域として扱える。境界だけが残れば、その境界と新点の接続で再三角形化できる。

### 後処理

全点挿入後、スーパー三角形に接続している三角形を除去する。利用者に返す頂点番号も、スーパー三角形分を除いたものへ詰め直す。

## 制約辺の回復

通常の Delaunay 分割だけでは、境界や穴を表す辺が分割の辺として残るとは限らない。たとえば頂点列だけを見て三角形分割すると、境界として残したい線分が、別の辺で横切られた形になることがある。そこで、点挿入後に制約辺を回復する。

ここでの目的は、Delaunay 性を完全に保ったまま全てを処理することではない。優先されるのは、指定された制約辺を分割の辺として必ず残すことである。そのため回復段階では、局所的な Delaunay 性よりも境界忠実性が前に出る。

また実装は、制約辺 $(u,v)$ の途中に既存頂点が共線に並んでいる場合、まず

$$
u = w_0,\; w_1,\; \dots,\; w_k = v
$$

という頂点列へ分割して、各区間 $(w_i, w_{i+1})$ を個別に回復する。この処理により、既存頂点をまたぐ長い制約辺でも局所操作へ分解できる。

### 辺フリップ

隣接する 2 つの三角形が凸四角形をなすとき、共有対角線をもう一方の対角線へ置き換えられる。この操作が辺フリップである。

```text
    c                    c
   /|\                  / \
  / | \                /   \
 /  |  \              /     \
a   |   b    -->     a-------b
 \  |  /              \     /
  \ | /                \   /
   \|/                  \ /
    d                    d

共有対角線 cd          共有対角線 ab
```

図の左では 2 つの三角形が対角線 $cd$ を共有している。四角形 $a$-$c$-$b$-$d$ が凸であれば、この対角線を $ab$ へ置き換えられる。これが辺フリップである。

制約辺を横切る辺があれば、フリップを繰り返して交差を解消できる。凸でない場合はフリップできないので、局所形状を見ながら進める必要がある。したがって回復は、交差辺を見つけて機械的に反転するだけではなく、「いま反転してよい局所配置か」を判定しながら進む操作である。

### 回復の流れ

制約辺 $e=(u,v)$ がまだ分割内に無いとき、考え方は「$u$ から $v$ へ引きたい線分を邪魔している辺を順にどかす」である。

制約辺 $e=(u,v)$ がまだ分割内に無ければ

1. $e$ と交差する辺を探す
2. フリップ可能な交差辺を反転する
3. 交差が消えるまで繰り返す
4. 最終的に $e$ 自体を分割の辺として得る

という流れになる。

回復の見通しは、「$e$ を横切る対角線を 1 本ずつどかして、最終的に $e$ 自身が対角線として現れるまで局所再配線を続ける」と表せる。通常の Delaunay 分割では $u$ と $v$ が互いに見えていても、その間に別の辺が横切ることがある。回復段階は、その交差を局所フリップで押しのけていく過程である。

実装上は、交差辺のうち

- 端点が制約辺の端点と一致しない
- それ自体が既存の制約辺ではない
- 隣接 2 三角形を持ち、フリップ候補になれる

ものを探して 1 本ずつ処理する。局所フリップが進められなくなると、制約辺回復は失敗として扱う。

回復後は、その制約辺をまたいで通常の Delaunay 改善を行わない。したがって最終結果は「自由 Delaunay 分割」ではなく、「制約辺を保持したうえで内部を可能な限り Delaunay に保った分割」である。

## 3D 判定

`orient3d` は 4 点が張る四面体の符号つき体積を返す。`insphere` は 5 点に対する外接球判定である。これらは 2D CDT 本体ではなく、厳密幾何判定 API の一部として公開している。

重要なのは、3D 側でも本質は同じという点である。向き判定と球内判定の符号を退化近傍で正しく返せることが、上位アルゴリズムの正しさを支える。
